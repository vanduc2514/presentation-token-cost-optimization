## How to Build

```
npm run build    # renders both language versions into output/
npm run preview  # opens output/index.html in the browser
```

Requires node 20 (pinned via `mise.toml`). Run `mise install` if switching environments.

### Output files

| File | Language |
|---|---|
| `output/index.html` | Vietnamese (default) |
| `output/index.en.html` | English |

Each file includes a floating language-switcher button linking to the other version.

### Remote Control

A mobile remote control is included. Open `output/remote.html` on any phone to control slide navigation and view speaker notes in real time via ntfy.sh relay.

## Presentation Content Rules

- **Slide content is short and punchy** — audience-facing, not speaker notes
- **Speaker notes are in HTML comments** — verbose, narrative, story-driven
- **No live demo** — examples and diagrams only
- **Never edit `output/index.html` or `output/index.en.html` directly** — they are generated by `build.cjs`

See `.agents/skills/markpress-content/SKILL.md` for writing and editing slides.

## Language Versions

The presentation exists in two languages. **Always keep both in sync** when editing content.

| File | Language | Audience |
|---|---|---|
| `slides/presentation.en.md` | English | International / default |
| `slides/presentation.vi.md` | Vietnamese | Vietnamese-speaking audiences |

Both files share the same slide structure, layout positions, and code examples. Translate content; never change slide order or `<!--slide-attr ...-->` positions between versions.

## Theming Guide

All theme customization is done in `build.cjs` — edit the `FONTS`, `CUSTOM CSS`, and `PER-SLIDE OVERRIDES` sections.

See `.agents/skills/markpress-styling/SKILL.md` for the full reference — it covers colors, fonts, background, slide card rules, performance constraints, and per-slide overrides.

### GitHub Badge — Required

The `githubBadge` constant in `build.cjs` injects a fixed GitHub icon into every slide. **This icon must always be present.** When changing the visual theme:

- You may change `.gh-badge` CSS (size, position, colors, opacity)
- You must NOT remove the `githubBadge` const or its injection (`.replace('<body>', ...)`)
- The `href` in `githubBadge` points to the presenter's GitHub profile — do not change it unless asked

## README Customization Guide

When customizing this repository for a specific talk, update `README.md` to include:

1. **Title and subtitle** — the talk name and one-sentence description
2. **Background section** — context for the talk, audience, and key ideas
3. **Render instructions** — keep the same build commands, update any prerequisites
4. **Talk outline** (optional) — a brief summary of each section
5. **Speaker bio** (optional) — a short bio and links

Remove or replace the "Use This Template" section once you've forked the template.

## Workflow

Always follow these steps when changing the presentation:

1. Plan the change
2. Make the change in `slides/presentation.en.md` AND `slides/presentation.vi.md`, or `build.cjs`
3. Run `npm run build` to render into HTML
4. Verify with visual QA — open `output/index.html` and `output/index.en.html` in the browser and check every slide
5. (Optional) If working in a cloud environment or commenting on a PR, include 3-5 screenshots
